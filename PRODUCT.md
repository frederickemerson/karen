# Karen

Karen is a terminal-first OpenCode distribution for developers who want agent power with public accountability. It keeps OpenCode's provider, model, MCP, session, and tool ecosystem, then adds prompt judgment, shame records, code-read quizzes, streaks, rewards, and rollback pressure.

Register: product

Users:
- Agent-heavy developers who work in the terminal and want a stronger feedback loop than a plain chat box.
- Builders who enjoy competitive, memorable, slightly hostile UX as long as it is useful.
- Teams that want evidence that developers can explain agent-generated code before it is committed.

Product purpose:
- Make `karen` the one-stop terminal interface for OpenCode-backed coding work.
- Block vague prompts before expensive or risky execution.
- Force a code comprehension check before commits are kept.
- Publish bad prompt and quiz failures to a feed/profile system.

Tone:
- Judgmental, fast, funny, and concrete.
- Mean about sloppy prompts, never careless with secrets or private code.
- Terminal-native, arcade-like, and high signal.

Anti-references:
- Generic SaaS dashboard softness.
- Chatbot wrapper with a logo.
- Shame that leaks customer data, secrets, or proprietary diffs.
- Dark-blue developer tool sameness.

Strategic principles:
- The terminal is the product. The web surface is the scoreboard.
- OpenCode remains the engine. Karen owns the ritual and policy.
- Every blocker explains the charge and the rewrite.
- Every fun effect must preserve flow speed.
